# noissh Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax. Every component is built TDD: failing test → minimal code → green → refactor → commit.

**Goal:** A mosh-resilient, SSH-rich remote shell built entirely on the Noise Protocol Framework, in Rust.

**Architecture:** A Cargo workspace of focused library crates (each independently testable, pure-logic where possible) plus two binaries (`noissh` client, `noisshd` server). Noise core wraps `snow` and is pure bytes-in/bytes-out. The transport layer adds a wire frame format (datagram + stream classes from day one), cryptographic session id, roaming, anti-replay, an input-reliability channel, and a v2 reliable stream multiplexer. The terminal model is an authoritative emulator + latest-wins screen diff. Predictive echo is client-only. Auth handles known_hosts (TOFU) and authorized_keys. Privsep/PAM/setuid is Linux-gated; a portable local-login PTY path is the runtime-tested one.

**Tech Stack:** Rust, `snow` (Noise), `vte` (terminal parser), `nix` (pty/fork/setuid/signals), `mio`/std UDP, `base64`, `rand`, `pam` (linux only).

---

## Workspace layout

```
crates/
  noise-core/   # snow wrapper: XX handshake + stateless transport AEAD
  wire/         # frame codec (datagram + stream frame classes), varint
  transport/    # session id, roaming, anti-replay, input-reliability, stream mux, congestion
  term/         # authoritative emulator (grid/cursor/modes) + latest-wins diff
  predict/      # predictive echo engine + reconciliation
  auth/         # known_hosts (TOFU), authorized_keys, x25519 key (de)serialization
  pty/          # portable PTY alloc + local login; linux privsep+PAM behind cfg
  proto/        # control-channel messages, handshake glue, session wiring shared by both ends
bin: src/bin/noissh.rs (client), src/bin/noisshd.rs (server)
```

## Build sequence (TDD per task; commit after each green)

### Task 1 — workspace + `wire` frame codec
- Varint encode/decode (u64 LEB128) round-trip + property test.
- `Frame` enum: `Ack{seq}`, `Input{offset, data}`, `StateDiff{seq, base, data}`, `Resize{cols,rows}`, `Ping{stamp}`, `Pong{stamp}`, `StreamOpen{id, kind, meta}`, `StreamData{id, offset, data, fin}`, `StreamAck{id, ack}`, `StreamClose{id, status}`, `StreamReset{id}`, `Control{data}`.
- `encode_frames(&[Frame]) -> Vec<u8>` / `decode_frames(&[u8]) -> Result<Vec<Frame>>`.
- Tests: round-trip every variant; truncated/garbage input errors cleanly (fuzz-style corpus). apply property test.

### Task 2 — `noise-core`
- `generate_keypair()`, `Keypair{private,public}` serialize/parse.
- `Handshake::initiator/responder(local_keys)`, `write_message`/`read_message`, `is_handshake_finished`, `remote_static()`, `into_transport() -> Session` (StatelessTransportState).
- `Session::encrypt(nonce,&[u8])`, `decrypt(nonce,&[u8])`.
- Tests: full XX handshake between two in-memory parties completes; transcript secrecy (ciphertext != plaintext); each side derives same remote_static; encrypt/decrypt round-trip at arbitrary nonces (out of order); tamper → decrypt error.

### Task 3 — `transport` datagram + roaming + anti-replay
- On-wire packet: `session_id(8) || nonce(8) || aead_ciphertext`. AD = session_id||nonce.
- `Endpoint`: maps session_id → SessionState{Session, peer_addr, replay_window, ...}. `recv(src_addr, bytes)` decrypts, on success updates peer_addr (roaming), enforces replay window, returns frames. `send(frames) -> bytes` with next nonce.
- Anti-replay sliding window (64-bit) per direction.
- Tests: encrypt→wire→decode round trip; replayed nonce rejected; too-old nonce rejected; out-of-window-future accepted+window slides; roaming: feeding a packet from a new src updates peer_addr only after auth succeeds, forged packet from new addr does NOT move peer_addr.

### Task 4 — `transport` input-reliability channel
- Client side: append-only input buffer; `pending() -> Frame::Input{first_unacked, bytes}`; `on_ack(seq)` advances. Server side: contiguous receiver; `ingest(Input) -> (newly_available_bytes, ack_seq)`.
- Tests: lossy/reordered Input frames reconstruct exact byte stream; duplicate inputs idempotent; ack advances; retransmit until acked.

### Task 5 — `term` emulator + diff
- `Cell{ch, fg, bg, flags}`, `Grid{rows,cols,cells,cursor,...}`, `Terminal` driving `vte::Parser`.
- Handle: printable + UTF-8, CR/LF/BS/TAB, CUP/CUU/CUD/CUF/CUB, ED/EL, SGR (bold/underline/reverse/fg/bg/reset), DECAWM wrap, scroll region (DECSTBM), index/RI, alt screen (1049), cursor show/hide, resize.
- `ScreenDiff` (latest-wins) encode/decode; `apply(diff)`; property test `apply(diff(a,b)) == b`.
- Tests: escape-sequence corpus (cursor moves, clears, colors, wrap, scroll, alt-screen); diff/apply property tests; resize reflow.

### Task 6 — `predict` predictive echo
- `Predictor`: on keystroke predict cell change + cursor advance when *safe* (printable, not in alt-screen/no-echo); track outstanding predictions with epoch; `reconcile(authoritative_diff)` confirms/abandons; `overlay(grid) -> grid` paints predictions (underline flag).
- Heuristics: don't predict control chars; suspend prediction in alt-screen; confidence ramp.
- Tests: replay recorded server diffs → predictions reconcile with zero residual; bad guess abandoned on conflicting diff; no prediction in password-like/alt-screen context.

### Task 7 — `auth`
- x25519 pubkey <-> `noissh-x25519 <base64>` text form.
- `AuthorizedKeys::parse/contains(pubkey)`.
- `KnownHosts::load/check_or_add(host, pubkey) -> Tofu{New|Match|Mismatch}`; persist.
- Tests: parse valid/invalid lines; TOFU new records, match passes, mismatch hard-fails; comments/blank lines ignored.

### Task 8 — `pty` + local login (portable) + linux privsep (gated)
- `openpty`, fork, setsid, set controlling tty, dup fds, exec `$SHELL`/login shell; set TERM, winsize; `set_winsize`.
- `LoginSession` trait: `spawn(user, env, winsize) -> PtyHandle`. `LocalLogin` (current user, portable, runtime-tested). `PrivsepLogin` (linux: PAM acct/open_session, setgid/initgroups/setuid; monitor fd-passing) behind `#[cfg(target_os="linux")]`.
- Tests (portable): spawn `/bin/echo`-style command in pty, read output; winsize set; resize SIGWINCH delivered.

### Task 9 — `transport` v2 reliable stream multiplexer
- Per-stream send/recv with offsets, ARQ retransmit of unacked, in-order delivery, flow-control window, FIN/close, reset.
- Tests: byte stream survives injected loss+reorder (ordered/lossless); flow control caps in-flight; multiple concurrent streams independent; close/exit-status delivered.

### Task 10 — `proto` wiring + control channel
- Control messages: `OpenShell{cols,rows,term,env}`, `Resize`, `ExitStatus`, `OpenForward{...}`, `ForwardData`, etc.
- `ServerSession` glue: handshake → authorize client key → open_session → pty → pump pty<->state-sync + input channel. `ClientSession` glue: handshake → TOFU → raw tty → predictive overlay → render.

### Task 11 — binaries + config + e2e
- `noisshd`: load static key (gen on first run), authorized_keys, bind UDP, accept sessions, spawn pty shell.
- `noissh user@host[:port]`: load/gen client key, known_hosts TOFU, connect, interactive session, resize propagation.
- Config files under `~/.config/noissh/`.
- **Integration/e2e tests (run on this machine):**
  - In-process e2e: client+server `Endpoint`s wired through a UDP shim, run a scripted shell, assert screen converges.
  - **Resilience harness:** UDP shim injecting loss/latency/reorder and *rewriting source address mid-session* → assert session survives and screen converges.
  - Real-process e2e: spawn `noisshd` + `noissh` over loopback UDP with a real PTY shell; send commands; assert output; change client source port; assert seamless resume.
  - Port-forward e2e: `-L` round-trips real TCP bytes through a stream, including across a forced source-address change.

### Task 12 — security tests
- Unauthorized client key rejected before any session/pty work.
- known_hosts mismatch aborts.
- Frame parser + handshake fuzz-style corpora (malformed input never panics).
